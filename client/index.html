UDP-like WebRTC Demo

Your connection protocol is {{protocol}}. If you're on TCP, this demo presumedly will not work and you'll just have TCP-like semantics instead.

The graph below represents each sent packet. A packet that got a "pong" response will appear in green. A packet that did not will show in red.

Once {{maxPackets}} packets have been sent, the server will be queried to see how many packets it actually received. This number may differ from the number of pongs received, since either part of the connection could fail.

  • Sent pings: {{pingCount}}
  • Received pongs: {{pongCount}}
  • Out of order pongs: {{outOfOrderCount}}

Server received pings: {{pingsReceived}} ({{pingsReceived / pingCount}})
